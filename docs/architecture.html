Architecture

SOARM Studio is intentionally thin. It owns workflow composition, local role assignment, camera preview, recording sidecars, and a small web surface. Motor I/O, motion, safety, and calibration stay in soarm-sdk.

Runtime path text 
leader arm -> 30 Hz teleop loop -> SDK direct joint stream -> follower arm
                         latest-frame cameras -> synchronized samples -> LeRobot-v3-compatible files
Module Responsibility
hardware.ports Serial discovery and SOARM bus probing.
hardware.cameras USB camera inventory, OpenCV preview, blocking camera adapters, and latest-frame wrappers for non-blocking teleop capture.
assignment Writes local leader/follower and wrist/third-person roles.
hardware.runtime Owns connected arms and cameras for check, teleop, and record.
recording Runs episodes and writes local dataset sidecars.
datasets LeRobot-v3-compatible local writer and inspection tools.
