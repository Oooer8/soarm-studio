Step 2 Calibration

Calibration captures physical zero first, then continuously records joint min/max while you move the arm by hand. The program disables torque and reads encoders; it does not command motion.

For the underlying model, see Calibration principles.

Before you calibrate

Run readiness check after Step 1 hardware setup. Treat this as pre-work for calibration, not as one of the interactive calibration commands.

Prerequisite readiness check zsh 
soarm-studio check --config configs/session.yaml --overwrite

Calibration commands

Calibrate one arm at a time. Start with the leader, finish the prompt-driven flow, then repeat the same process for the follower.

Interactive calibration zsh 
soarm-studio calibrate --config configs/session.yaml --role leader
soarm-studio calibrate --config configs/session.yaml --role follower

Leader terminal walkthrough

After you start leader calibration, the command opens configs/arms/leader.yaml, records pre-calibration diagnostics, disables torque, and pauses at the first prompt.

Start leader calibration terminal 
$ soarm-studio calibrate --config configs/session.yaml --role leader
Place every joint in the physical zero pose, then press Enter to record zero ticks:

At this prompt, move the leader arm by hand into the physical zero pose. Align every joint deliberately, keep the arm still, then press Enter once. The SDK reads the raw encoder ticks and saves them as temporary zero_tick values for the sweep.

Range sweep prompt terminal 
Freely sweep the full range of motion for each joint. Press Enter when done:

Now slowly move all leader joints through their safe usable range by hand. Cover shoulder, elbow, wrist, and gripper travel without forcing mechanical stops. When every joint has been exercised, stop moving the arm and press Enter. The SDK computes the observed min/max range for each joint and writes the calibrated values back to configs/arms/leader.yaml.

Completion report

When calibration finishes, SOARM Studio prints a JSON result. The exact diagnostics depend on your hardware, but a successful run has ok: true, the role, the config path, and a report with zero ticks, sweep ranges, soft limits, and post-check diagnostics.

Representative completed output json 
{
  "ok": true,
  "role": "leader",
  "results": [
    {
      "role": "leader",
      "ok": true,
      "config": "configs/arms/leader.yaml",
      "report": [
        "Pre-calibration status:",
        "...",
        "Calibration step 1/4: disabling servo torque for manual movement.",
        "Calibration step 2/4: recording the physical zero pose.",
        "  Zero ticks: shoulder_pan=2092, shoulder_lift=2053, ...",
        "Calibration step 3/4: dynamic sweep of all joints.",
        "  Noise filter: median window=3 | Limit margin: 0 ticks | Motion threshold: 100 ticks",
        "  shoulder_pan: direction=+1 | raw=[770, 3410] ticks (range=2640) | safe=[-2.0325, 2.0586] rad",
        "...",
        "Calibration step 4/4: saved calibrated config to configs/arms/leader.yaml.",
        "Post-calibration status:",
        "..."
      ]
    }
  ]
}

Repeat and verify

Repeat the same terminal flow with --role follower. After both arms are calibrated, run readiness check again as the handoff into teleoperation.

Post-calibration verification zsh 
soarm-studio check --config configs/session.yaml --overwrite
!
Under-excited joints

A joint with less than 100 ticks of observed motion is reported as under-excited. Re-run calibration for that arm and move that joint through more of its safe range.
