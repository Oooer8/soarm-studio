SOARM Studio documentation

SOARM Studio is the local control and data-capture workspace for SOARM arms. It turns a physical leader/follower setup into a repeatable workflow for hardware bring-up, readiness checks, calibration, low-latency teleoperation, dataset recording, and recording-quality review.

i
Positioning

Studio makes the SOARM hardware trustworthy on the table. LeRobot remains the broader robot-learning ecosystem for standard datasets, training, policy deployment, Hugging Face Hub workflows, and sharing.

Position and foundation

Ecosystem role

Studio is the SOARM-specific application layer before data and policies enter the learning stack. It records toward LeRobot compatibility, but keeps hardware setup, operation, and local diagnostics close to the arm.

Implemented foundation

The core workflow covers device discovery, role binding, preflight checks, calibration, leader-to-follower teleoperation, latest-frame camera capture, synchronized recording, local dataset output, and per-episode quality summaries.

Studio and LeRobot

For SOARM body control and motion-control workflows, Studio is optimized for trustworthy local operation. LeRobot is optimized for the larger learning ecosystem. The two are complementary, but their strengths are different.

Dimension Stronger choice Why
Sample quality Studio Studio records real capture timing, camera latency and frame age, dropped or stale camera indicators, and quality sidecars. LeRobot provides the standard dataset ecosystem, but its general recording path is less focused on SOARM-specific timing accountability.
Control stability Studio for SOARM operation Studio owns the SOARM setup path: port assignment, preflight, calibration, runtime checks, and guarded local recording. LeRobot is stronger as a mature cross-robot data and policy stack.
Real-time behavior Studio Studio separates target updates from the fixed-rate arm output stream and avoids blocking the control loop on fresh camera capture. LeRobot keeps observation, action, sending, recording, and display closer to one main loop, so command cadence depends more on full-loop load.
Training ecosystem LeRobot LeRobot remains the better place for standardized dataset tooling, policy training, policy deployment, Hub workflows, and community conventions. Studio is designed to feed that ecosystem with better local SOARM sessions.
