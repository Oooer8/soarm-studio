SOARM Studio documentation

SOARM Studio exists beside LeRobot for the parts of SOARM operation where a robot-specific workflow matters most: hardware bring-up, port assignment, readiness checks, calibration, low-latency leader-to-follower teleoperation, and local recording quality.

i
Studio and LeRobot

Studio makes the arm trustworthy on the table. LeRobot remains the target ecosystem for dataset compatibility, training, policy deployment, and sharing.

Where Studio helps

Studio keeps SOARM setup explicit and local: scan devices, bind leader and follower roles, run preflight checks, calibrate each arm, then stream leader targets through the SDK-owned joint controller while recording synchronized samples and quality sidecars.

Where LeRobot helps

LeRobot is the broader robot-learning stack: standard dataset tooling, Hugging Face Hub workflows, policy training, policy rollout, visualization, and community hardware conventions. Studio records toward that ecosystem instead of replacing it.
