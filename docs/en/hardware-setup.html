Hardware setup

Hardware setup binds the two visible arms and two camera streams on this machine into configs/session.yaml. Calibration, teleoperation, and recording all use this machine-local config after setup.

Before you start:

  • Both arms are assembled, and the motors use the SOARM default IDs 1-6, or you have a custom --ids list ready.
  • Arm power and the servo buses are ready, the USB serial cables can be connected one arm at a time, and the motors are readable.
  • The wrist and third-person cameras are connected over USB and placed where they will be used.
  • The workspace is clear, so the arms can be powered safely during preview and checks.

Scan hardware

First list the arm serial ports and USB cameras visible to this machine. The serial port serial / location fields and camera USB metadata help with role assignment and troubleshooting later.

CLI 
soarm-studio scan --include-system

Configure arms

  1. Identify roles by hot-plugging

    Keep both arms connected for a baseline bus probe, then hot-plug one arm connection at a time and repeat the probe. Compare arm_ports: the device that disappears and returns belongs to the arm you hot-plugged.

    1. Run the probe below with both arms connected and record both device and online_ids values.
    2. Hot-plug the leader connection and probe again; compare the lists to identify the leader port.
    3. Reconnect the leader, hot-plug the follower, and probe again to identify the follower port.
    4. Reconnect both arms and probe once more to confirm IDs 1-6 are online on both ports.
    CLI 
    soarm-studio scan --probe-arms
    Show port comparison example 
    Baseline probe:
  /dev/ttyACM0  online_ids=[1,2,3,4,5,6]
  /dev/ttyACM1  online_ids=[1,2,3,4,5,6]

Probe after hot-plugging the leader:
  /dev/ttyACM1  online_ids=[1,2,3,4,5,6]

Confirmed leader port:
  /dev/ttyACM0

    Repeat the same comparison for the follower. Do not infer roles from port sort or insertion order; after reconnecting both arms, use the same command for the final bus check.

    Show expected output 
    {
  "summary": {
    "ok": true,
    "probed_ports": 2,
    "online_ports": 2
  },
  "arm_ports": [
    {
      "device": "/dev/ttyACM1",
      "ok": true,
      "expected_ids": [1, 2, 3, 4, 5, 6],
      "online_ids": [1, 2, 3, 4, 5, 6]
    },
    {
      "device": "/dev/ttyACM0",
      "ok": true,
      "expected_ids": [1, 2, 3, 4, 5, 6],
      "online_ids": [1, 2, 3, 4, 5, 6]
    }
  ],
  "next_steps": [
    "Use the verified devices in setup arms as leader/follower ports."
  ]
}

    If online_ids has fewer entries than expected_ids, check power, bus wiring, motor IDs, and baud rate first. If you changed motor IDs, pass them with --ids.

  2. Write arm roles

    Write the two ports confirmed by the hot-plug scans into the local runtime config. In this example, 5A7C121905 is the leader and 5A7C119035 is the follower.

    CLI 
    soarm-studio setup arms \
  --leader-port /dev/ttyACM0 \
  --follower-port /dev/ttyACM1
    Show expected output 
    {
  "session_config": "configs/session.yaml",
  "arm_configs": {
    "leader": "configs/arms/leader.yaml",
    "follower": "configs/arms/follower.yaml"
  },
  "warnings": []
}

Configure cameras

  1. Preview cameras

    Preview multiple OpenCV indexes at once, save frames, then use the image content to identify wrist and third-person views. On Ubuntu these indexes map to V4L2 devices such as /dev/video0; after unplugging or changing hubs, preview again instead of trusting the old index.

    CLI 
    soarm-studio scan --preview-cameras \
  --camera-indices 0,1,2,3 \
  --backend v4l2 \
  --output-dir previews/cameras

  2. Write camera roles

    Write the confirmed wrist and third-person indexes into configs/session.yaml. OpenCV indexes can change after unplugging, rebooting, or moving to another USB hub, so preview again after hardware placement changes.

    CLI 
    soarm-studio setup cameras \
  --backend v4l2 \
  --wrist-index 1 \
  --third-person-index 0
    Show expected output 
    {
  "session_config": "configs/session.yaml",
  "assigned": {

    }
  },
  "warnings": [
    "OpenCV indexes can change; confirm roles with preview. USB metadata is saved only when the mapping is unambiguous."
  ]
}

Check readiness and write outputs

Finally, reopen both servo buses and both cameras through the generated session config, then confirm bindings and preflight pass. --overwrite refreshes check outputs so calibration can continue from the same config.

soarm-studio check --config configs/session.yaml --overwrite
