Dataset recording

Before writing a dataset, run teleop without recording and confirm joint directions, usable ranges, gripper behavior, and loop frequency. Start episode capture only after that check passes.

Validate teleop before recording

Complete Arm calibration and run check first. Clear the follower workspace, place both arms near safe mid-range poses, then run a 30-second free test. --free-test repeats readiness checks before the control loop starts.

CLI 
soarm-studio teleop \
  --config configs/session.yaml \
  --free-test \
  --seconds 30

  1. Once the command starts, hold the leader and slowly move the shoulder, elbow, and wrist joints one at a time, then open and close the gripper. Confirm the follower moves continuously in the same direction without hitting software limits before trying combined motion.

  2. If the follower reverses direction, jumps, approaches a mechanical hard stop, or develops unusual lag, press Ctrl-C immediately and do not record. Recheck role bindings, calibration ranges, and check output first.

  3. A healthy run ends in TELEOP_READY, with observed_hz close to the configured target_hz. Proceed to recording only after this passes.

Run teleop diagnostics when needed

If loop frequency is low or motion lags, start with debug output. Add --follower-readback-every 10 only when periodic post-command follower state is needed. See Teleoperation output for metric definitions.

CLI 
soarm-studio teleop \
  --config configs/session.yaml \
  --free-test \
  --seconds 60 \
  --debug

soarm-studio teleop \
  --config configs/session.yaml \
  --free-test \
  --seconds 60 \
  --debug \
  --follower-readback-every 10

Record the dataset

Recording writes a local LeRobot Dataset v3 dataset plus SOARM Studio sidecars describing the hardware session and sync quality. Start with one short episode to verify task text, images, joint data, and output paths before scaling capture.

CLI 
soarm-studio record \
  --config configs/session.yaml \
  --episodes 1 \
  --warmup 1 \
  --seconds 10 \
  --task "pick object" \
  --overwrite

Capture timing

During recording, the live loop only records joint state, action, and monotonic timestamps; image synchronization is handled after the episode. The first episode's --warmup and capture segment share one TeleopLoop, so Studio does not run a second sync_start or restart the follower stream after warmup. Cameras that support history start saving background frames at the episode boundary. Before writing the dataset, Studio matches each control sample to the nearest camera frame by monotonic_time_ns. 

camera threads:
  camera.read() @ camera.fps -> latest frame

warmup:
  same TeleopLoop, no sample/history collection

episode start:
  enable episode camera history

Studio loop @ loop_hz:
  leader_read || follower_before_read
  -> action
  -> SDK direct stream update
  -> append ControlSample(joints, action, timestamp)

after episode:
  sample.timestamp -> nearest camera frame timestamp
  -> write LeRobot frame and quality sidecars

This means teleop debug camera_age:* reports latest-frame preview freshness, while recording quality reports the post-matching sample-to-image offset.

Normal recording only saves summary quality metrics. Add --debug to the record command when you need raw per-camera frame timestamps, frame intervals, and per-sample matching offsets; the sidecar is written to episodes/episode_000000/camera_timing.json.
