LeRobot Dataset v3

SOARM Studio records a local LeRobot Dataset v3 dataset. The format separates the episode-level API from the storage layer: low-dimensional state and action data go to Parquet, camera frames are encoded as MP4, and metadata describes schema, statistics, task text, and where each episode lives inside shared files.

This page is based on the Hugging Face LeRobotDataset v3.0 docs and the path templates in the current LeRobot source.

Format Parts

Part Purpose
Tabular data Frame-by-frame values such as timestamp, observation.state, and action are stored as Parquet files under data/.
Visual data Each camera becomes an observation.images.<camera> feature, with MP4 files stored under videos/.
Metadata meta/info.json stores schema, fps, version, and path templates; meta/stats.json stores normalization statistics; meta/episodes/ stores episode lengths, tasks, and file offsets.

Local Layout

SOARM Studio currently writes a compact local v3 layout. LeRobot v3 is designed so many episodes can be aggregated into larger Parquet/MP4 shards; SOARM starts with one local chunk for straightforward inspection and debugging. 

datasets/soarm/
  meta/
    info.json
    stats.json
    tasks.parquet
    episodes/chunk-000/file-000.parquet
    soarm_session.json
    sync_quality.json
  data/chunk-000/file-000.parquet
  videos/observation.images.<camera>/chunk-000/file-000.mp4
  episodes/episode_000000/quality.json
i
Current repo format check

src/soarm_studio/datasets/lerobot_v3/schema.py uses codebase_version=v3.0 and follows the current LeRobot source templates for meta/tasks.parquet, meta/episodes/chunk-000/file-000.parquet, data/chunk-000/file-000.parquet, and videos/{video_key}/chunk-000/file-000.mp4. meta/soarm_session.json, meta/sync_quality.json, and episode quality files are SOARM-specific sidecars.

Feature Meaning

Feature SOARM meaning
observation.state Follower joint positions read before sending the action, ordered by the joints list in configs/session.yaml.
action Target joint positions sent to the follower direct joint stream for the current frame.
timestamp Episode-relative time. The first frame is 0 seconds, and later frames use the control loop's monotonic clock.
task_index Integer task ID mapped to task text in meta/tasks.parquet.
observation.images.* Enabled wrist, third-person, or other camera video features stored under videos/.
