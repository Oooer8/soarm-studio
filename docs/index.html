SOARM Studio documentation

SOARM Studio is the application layer for dual SOARM bring-up, calibration, teleoperation, and local data collection. It keeps the user workflow clear while delegating low-level arm behavior to soarm-sdk.

i
README vs docs

The README is a short project entry point. This page is the detailed operating manual for installation, concepts, hardware setup, calibration, recording, media examples, and troubleshooting.

Recommended command path

Use scan, setup, check, calibrate, teleop, and record. Older duplicate command shapes were removed from the public CLI.

One generated runtime config

configs/session.yaml is the local runtime config for one machine. Example configs remain templates under configs/sessions/.

Installation

The project expects a sibling ../soarm checkout for the SDK. The conda environment installs both packages editable so CLI changes are immediately available.

Install the environment zsh 
conda env create -f environment.yml
conda activate soarm-studio
Validate the local install zsh 
soarm-studio --help
soarm-studio check --config configs/sessions/mock.yaml --overwrite

Quickstart

The full hardware path is staged. Do not skip directly to teleop or recording. Bring up hardware first, calibrate next, then do free motion tests and dataset capture.

End-to-end command outline zsh 
soarm-studio scan --include-system
soarm-studio setup arms \
  --leader-port /dev/cu.usbmodemLEADER \
  --follower-port /dev/cu.usbmodemFOLLOWER \
  --base-arm-config ../soarm/configs/soarm.yaml
soarm-studio scan --preview-cameras --camera-indices 0,1,2,3 --backend avfoundation
soarm-studio setup cameras --backend avfoundation --wrist-index 1 --third-person-index 0
soarm-studio check --overwrite
soarm-studio calibrate --role leader
soarm-studio calibrate --role follower
soarm-studio check --overwrite
soarm-studio teleop --free-test --seconds 5
soarm-studio record --episodes 1 --warmup 1 --seconds 10 --task "pick object" --overwrite

Architecture

SOARM Studio is intentionally thin. It owns workflow composition, local role assignment, camera preview, recording sidecars, and a small web surface. Motor I/O, motion, safety, and calibration stay in soarm-sdk.

Runtime path text 
leader arm -> teleop loop -> safety clipping -> follower arm
                         -> recorder -> LeRobot-v3-compatible files
Module Responsibility
hardware.ports Serial discovery and SOARM bus probing.
hardware.cameras USB camera inventory, OpenCV preview, and capture adapters.
assignment Writes local leader/follower and wrist/third-person roles.
hardware.runtime Owns connected arms and cameras for check, teleop, and record.
recording Runs episodes and writes local dataset sidecars.
datasets LeRobot-v3-compatible local writer and inspection tools.

Configuration

Setup writes one runtime config and two per-arm configs. These files are ignored because they depend on your USB ports, camera indexes, and calibrated limits.

Generated files text 
configs/session.yaml
configs/arms/leader.yaml
configs/arms/follower.yaml
!
Do not treat examples as runtime state

Files under configs/sessions/ are templates. Use configs/session.yaml for real runs after setup.

Publishing on GitHub Pages

This documentation site is static and can be served directly from the repository's docs/ directory.

  1. Open the GitHub repository settings for Oooer8/soarm-studio.
  2. In Pages, choose Deploy from a branch.
  3. Set the branch to main and the folder to /docs.
  4. Keep docs/.nojekyll so nested assets are served without Jekyll processing.

Future photos, screenshots, and videos should stay under docs/assets/. Runtime outputs such as camera previews, datasets, and calibration captures remain outside the docs tree unless a curated copy is intentionally added.

Step 1 Hardware setup

Step 1 discovers local hardware, assigns roles, and writes the runtime config. It stops before calibration or motion.

Scan hardware

List serial ports and USB camera inventory.

soarm-studio scan --include-system

Probe arms

Probe one bus at a time when ports are ambiguous.

soarm-studio scan --probe-arms --arm-config ../soarm/configs/soarm.yaml

Setup arm roles

Save explicit leader and follower ports.

soarm-studio setup arms \
  --leader-port /dev/cu.usbmodemLEADER \
  --follower-port /dev/cu.usbmodemFOLLOWER \
  --base-arm-config ../soarm/configs/soarm.yaml

Preview cameras

Inspect saved frames before assigning wrist and third-person.

soarm-studio scan --preview-cameras \
  --camera-indices 0,1,2,3 \
  --backend avfoundation \
  --output-dir previews/cameras

Setup camera roles

Save indexes confirmed from preview images.

soarm-studio setup cameras \
  --backend avfoundation \
  --wrist-index 1 \
  --third-person-index 0

Check readiness

Verify bindings and run live readiness checks.

soarm-studio check --config configs/session.yaml --overwrite

Step 2 Calibration

Calibration captures physical zero first, then continuously records joint min/max while you move the arm by hand. The program disables torque and reads encoders; it does not command motion.

Recommended calibration sequence zsh 
soarm-studio check --config configs/session.yaml --overwrite
soarm-studio calibrate --config configs/session.yaml --role leader
soarm-studio calibrate --config configs/session.yaml --role follower
soarm-studio check --config configs/session.yaml --overwrite

What happens internally

  1. The Studio wrapper opens the selected role config.
  2. The SDK records pre-calibration diagnostics.
  3. Servo torque is disabled for manual movement.
  4. You place the arm at physical zero and press Enter.
  5. The SDK captures raw encoder ticks as zero_tick.
  6. You sweep all joints through safe motion by hand.
  7. The SDK samples at about 50 Hz and tracks raw min/max ticks.
  8. The SDK writes updated zero_tick, direction, min_rad, and max_rad.
!
Under-excited joints

A joint with less than 100 ticks of observed motion is reported as under-excited. Re-run calibration for that arm and move that joint through more of its safe range.

Teleoperation

Teleop reads the leader arm, clips follower targets through safety, streams commands to the follower, and can emit control samples for recording. 

soarm-studio teleop --config configs/session.yaml --free-test --seconds 5

Use --free-test after calibration. It runs readiness checks before starting the loop.

Recording

Recording writes LeRobot-v3-compatible local data plus SOARM Studio sidecars that describe the hardware session and sync quality. 

soarm-studio record \
  --config configs/session.yaml \
  --episodes 1 \
  --warmup 1 \
  --seconds 10 \
  --task "pick object" \
  --overwrite
Output Purpose
meta/soarm_session.json Session, config, and hardware binding snapshot.
meta/sync_quality.json Dataset-level timing and capture quality summary.
episodes/episode_000000/quality.json Per-episode timing and frame quality metrics.

CLI reference

The public CLI is intentionally small.

Command Use
scan Find ports, cameras, and optional previews.
setup Save arm and camera roles into runtime config.
check Verify bindings and readiness.
calibrate Run interactive zero and continuous range capture.
teleop Run leader-to-follower control.
record Record local episodes and sidecars.
dataset Inspect, validate, or export local datasets.
web Run the local browser UI.

Troubleshooting

Camera preview fails

On macOS, grant Camera permission to the terminal application. Then re-run scan --preview-cameras.

Wrong camera role

OpenCV indexes can change. Re-capture previews and re-run setup cameras.

Serial port denied

Verify the terminal has permission to open /dev/cu.* and no other process owns the same port.

Calibration feels wrong

Re-run one arm at a time. Capture zero carefully, then sweep every joint through the safe range you want available.
