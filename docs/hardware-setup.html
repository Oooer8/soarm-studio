检查并写入配置

第一步先把本机可见的两台机械臂和两路摄像头绑定到 configs/session.yaml。完成后，标定、 遥操作和录制都使用这份本机配置。

开始前确认：

  • 两台机械臂已经装好，电机使用 SOARM 默认 ID 1-6，或准备好自定义 --ids。
  • 机械臂电源、总线和 USB 串口都已连接，电机处于可读取状态。
  • wrist 与 third-person 摄像头都已接入 USB，并放到实际使用位置。
  • 操作区域已清空，后续预览和检查时机械臂可以安全上电。

扫描硬件

先列出当前机器能看到的机械臂串口和 USB 摄像头。串口的 serial / location 以及摄像头的 USB 信息会帮助你在后面做角色绑定和排查。

CLI 
soarm-studio scan --include-system

配置机械臂

  1. 探测机械臂端口

    在写入角色前，先让 CLI 逐个串口读取默认 ID 1-6，确认两个端口都能连上完整的电机总线。

    CLI 
    soarm-studio scan --probe-arms
    查看预期输出 
    {
  "summary": {
    "ok": true,
    "probed_ports": 2,
    "online_ports": 2
  },
  "arm_ports": [
    {
      "device": "/dev/cu.usbmodem5A7C1190351",
      "ok": true,
      "expected_ids": [1, 2, 3, 4, 5, 6],
      "online_ids": [1, 2, 3, 4, 5, 6]
    },
    {
      "device": "/dev/cu.usbmodem5A7C1219051",
      "ok": true,
      "expected_ids": [1, 2, 3, 4, 5, 6],
      "online_ids": [1, 2, 3, 4, 5, 6]
    }
  ],
  "next_steps": [
    "Use the verified devices in setup arms as leader/follower ports."
  ]
}

    如果 online_ids 少于 expected_ids，先检查电源、总线接线、电机 ID 和波特率；如果你改过电机编号，用 --ids 指定。

  2. 写入机械臂角色

    根据实际摆放决定 leader 和 follower，把确认过的两个串口写入本机运行配置。下面示例中 5A7C121905 是 leader，5A7C119035 是 follower。

    CLI 
    soarm-studio setup arms \
  --leader-port /dev/cu.usbmodem5A7C1219051 \
  --follower-port /dev/cu.usbmodem5A7C1190351
    查看预期输出 
    {
  "session_config": "configs/session.yaml",
  "arm_configs": {
    "leader": "configs/arms/leader.yaml",
    "follower": "configs/arms/follower.yaml"
  },
  "warnings": []
}

配置摄像头

  1. 预览摄像头

    一次预览多个 OpenCV index，保存图片后用画面内容确认 wrist 和 third-person。 macOS 上相同型号摄像头可能同名，所以不要只靠设备名判断。

    CLI 
    soarm-studio scan --preview-cameras \
  --camera-indices 0,1,2,3 \
  --backend avfoundation \
  --output-dir previews/cameras
    查看预期输出

    "previewed": 4,
    "saved_frames": 4

  2. 写入摄像头角色

    把确认过的 wrist 和 third-person index 写入 configs/session.yaml。OpenCV index 可能随插拔、重启或换 USB hub 改变，硬件位置变化后要重新预览。

    CLI 
    soarm-studio setup cameras \
  --backend avfoundation \
  --wrist-index 1 \
  --third-person-index 0
    查看预期输出 
    {
  "session_config": "configs/session.yaml",
  "assigned": {
    "wrist": {
      "device": 1,
      "backend": "avfoundation",
      "width": 640,
      "height": 480,
      "fps": 30
    },
    "third_person": {
      "device": 0,
      "backend": "avfoundation",
      "width": 640,
      "height": 480,
      "fps": 30
    }
  },
  "warnings": [
    "OpenCV indexes can change; confirm roles with preview. USB metadata is saved only when the mapping is unambiguous."
  ]
}

检查并写入配置

最后用生成的 session 配置重新打开两条 servo bus 和两个摄像头，确认 bindings 和 preflight 都通过。 --overwrite 会刷新检查产物，方便后续标定继续使用。
