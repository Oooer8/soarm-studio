Publishing on GitHub Pages

This documentation site is static. The source lives in docs/ on main, and GitHub Actions publishes that directory as a Pages artifact.

  1. Open the GitHub repository settings for Oooer8/soarm-studio.
  2. In Pages, choose GitHub Actions as the source.
  3. The workflow at .github/workflows/pages.yml uploads docs/.
  4. The deploy job publishes the artifact to the github-pages environment.

Future photos, screenshots, and videos should stay under docs/assets/. Runtime outputs such as camera previews, datasets, and calibration captures remain outside the docs tree unless a curated copy is intentionally added.
