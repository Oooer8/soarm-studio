SOARM Studio 文档

SOARM Studio 放在 LeRobot 旁边，是为了补上 SOARM 运行中最需要本体化工作流的部分：硬件启动、串口分配、状态预检、机械臂标定、低延迟主从遥操作，以及本地录制质量检查。

i
Studio 与 LeRobot

Studio 负责让桌面上的机械臂先变得可信、可诊断、可稳定遥操作；LeRobot 仍然是数据集兼容、训练、策略部署和共享的目标生态。

Studio 的优势

Studio 把 SOARM 的本地流程做清楚：扫描设备，绑定 leader 和 follower，运行预检，分别标定两只机械臂，再把 leader 目标送入 SDK 持有的关节流控制器，同时记录同步样本和质量侧车。

LeRobot 的优势

LeRobot 是更大的机器人学习栈：标准数据集工具、Hugging Face Hub 工作流、策略训练、策略部署、可视化和社区硬件约定。Studio 面向这个生态录制数据，而不是替代它。
