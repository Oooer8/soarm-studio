SOARM Studio 文档

SOARM Studio 是面向 SOARM 机械臂本体运行和数据采集的本地工作台。它把 leader/follower 机械臂从硬件启动、状态预检、标定、低延迟遥操作，到数据集录制和质量检查，组织成一条可重复执行的工作流。

i
定位

Studio 负责让桌面上的 SOARM 机械臂先变得可信、可诊断、可稳定遥操作；LeRobot 仍然是标准数据集、策略训练、策略部署、Hugging Face Hub 工作流和共享协作的更大机器人学习生态。

生态定位和基础能力

生态中的定位

Studio 是数据和策略进入学习栈之前的 SOARM 专用应用层。它面向 LeRobot 兼容格式录制数据，但把硬件启动、运行、诊断和本地质量控制留在离机械臂最近的位置。

已经实现的基础能力

核心流程包括设备发现、角色绑定、预检、标定、leader-to-follower 遥操作、最新帧相机采集、同步录制、本地数据集输出，以及 episode 级别的质量摘要。

Studio 与 LeRobot

如果问题聚焦在 SOARM 机械臂的本体控制和运控层面，Studio 更偏向可信的本地运行；LeRobot 更偏向通用机器人学习生态。二者互补，但优势不在同一个层面。

维度 更适合 原因
采样质量 Studio Studio 记录真实采样时间、相机读取延迟、画面帧龄、丢帧或陈旧帧状态，并生成质量 sidecar。LeRobot 的优势是标准数据集生态，但通用录制链路对 SOARM 专用时序质量的表达更轻。
控制稳定性 SOARM 本体运行更适合 Studio Studio 覆盖 SOARM 的本地运行路径：端口分配、状态预检、机械臂标定、运行期检查和带保护的本地录制。LeRobot 更强的是跨机器人数据和策略生态的成熟度。
实时性 Studio Studio 将目标更新和固定频率的机械臂输出流解耦，并避免控制循环被相机新帧采集阻塞。LeRobot 的观测、动作、发送、录制和显示更接近一个主循环，动作节奏更容易受整条循环负载影响。
训练生态 LeRobot LeRobot 仍然更适合标准数据集工具、策略训练、策略部署、Hub 工作流和社区约定。Studio 的目标是为这个生态提供质量更可控的 SOARM 本地会话。
