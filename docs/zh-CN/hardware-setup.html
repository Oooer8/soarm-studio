硬件设置

硬件设置会把本机可见的两台机械臂和两路摄像头绑定到 configs/session.yaml。完成后，标定、 遥操作和录制都使用这份本机配置。

开始前确认：

  • 两台机械臂已经装好，电机使用 SOARM 默认 ID 1-6，或准备好自定义 --ids。
  • 机械臂电源和舵机总线已就绪，USB 串口线可以逐台插拔，电机处于可读取状态。
  • wrist 与 third-person 摄像头都已接入 USB，并放到实际使用位置。
  • 操作区域已清空，后续预览和检查时机械臂可以安全上电。

扫描硬件

先列出当前机器能看到的机械臂串口和 USB 摄像头。串口的 serial / location 以及摄像头的 USB 信息会帮助你在后面做角色绑定和排查。

CLI 
soarm-studio scan --include-system

配置机械臂

  1. 热插拔确认主从臂串口

    先让两台机械臂保持连接并完成一次总线基准探测，再逐台热插拔机械臂连接并重复探测。对比每次 arm_ports，消失后重新出现的 device 就是刚才热插拔的那台机械臂串口。

    1. 两台机械臂都连接时运行下面的探测，记录两个 device 及其 online_ids。
    2. 热插拔主臂连接并重新探测；对比前后列表，确认主臂串口。
    3. 主臂恢复连接后，热插拔从臂并重新探测；确认从臂串口。
    4. 重新连接两台机械臂，再探测一次，确认两个端口的 ID 1-6 都在线。
    CLI 
    soarm-studio scan --probe-arms
    查看端口对比示例 
    基准探测:
  /dev/ttyACM0  online_ids=[1,2,3,4,5,6]
  /dev/ttyACM1  online_ids=[1,2,3,4,5,6]

热插拔主臂后再次探测:
  /dev/ttyACM1  online_ids=[1,2,3,4,5,6]

确认主臂串口:
  /dev/ttyACM0

    从臂按相同方法确认。不要根据端口排序或插入顺序猜测角色；两台机械臂重新连接后，仍使用同一条命令完成最终总线复查。

    查看预期输出 
    {
  "summary": {
    "ok": true,
    "probed_ports": 2,
    "online_ports": 2
  },
  "arm_ports": [
    {
      "device": "/dev/ttyACM1",
      "ok": true,
      "expected_ids": [1, 2, 3, 4, 5, 6],
      "online_ids": [1, 2, 3, 4, 5, 6]
    },
    {
      "device": "/dev/ttyACM0",
      "ok": true,
      "expected_ids": [1, 2, 3, 4, 5, 6],
      "online_ids": [1, 2, 3, 4, 5, 6]
    }
  ],
  "next_steps": [
    "Use the verified devices in setup arms as leader/follower ports."
  ]
}

    如果 online_ids 少于 expected_ids，先检查电源、总线接线、电机 ID 和波特率；如果你改过电机编号，用 --ids 指定。

  2. 写入机械臂角色

    把热插拔扫描确认过的两个串口写入本机运行配置。下面示例中 5A7C121905 是 leader，5A7C119035 是 follower。

    CLI 
    soarm-studio setup arms \
  --leader-port /dev/ttyACM0 \
  --follower-port /dev/ttyACM1
    查看预期输出 
    {
  "session_config": "configs/session.yaml",
  "arm_configs": {
    "leader": "configs/arms/leader.yaml",
    "follower": "configs/arms/follower.yaml"
  },
  "warnings": []
}

配置摄像头

  1. 预览摄像头

    一次预览多个 OpenCV index，保存图片后用画面内容确认 wrist 和 third-person。 Ubuntu 上这些 index 对应 /dev/video0 这类 V4L2 设备；插拔或更换 USB hub 后要重新预览，不要沿用旧 index。

    CLI 
    soarm-studio scan --preview-cameras \
  --camera-indices 0,1,2,3 \
  --backend v4l2 \
  --fourcc MJPG \
  --output-dir previews/cameras
    查看预期输出

  2. 写入摄像头角色

    把确认过的 wrist 和 third-person index 写入 configs/session.yaml。OpenCV index 可能随插拔、重启或换 USB hub 改变，硬件位置变化后要重新预览。

    CLI 
    soarm-studio setup cameras \
  --backend v4l2 \
  --fourcc MJPG \
  --wrist-index 1 \
  --third-person-index 0
    查看预期输出 
    {

检查并写入配置

最后用生成的 session 配置重新打开两条 servo bus 和两个摄像头，确认 bindings 和 preflight 都通过。 --overwrite 会刷新检查产物，方便后续标定继续使用。

CLI 
soarm-studio check --config configs/session.yaml --overwrite
