遥操作

遥操作是在 SDK direct joint stream 外包裹的一层 30 Hz Studio 侧采样循环。Studio 读取主臂和从臂动作前状态，计算裁剪后的目标，然后把目标发布给 soarm-sdk。SDK 负责固定频率的从臂输出、最新目标写入、关节限位检查和超时保持。

具体上机顺序、主臂操作方法和诊断命令见数据集录制前的遥操作验证；本页只解释控制路径和输出指标。

技术方案

默认实时路径针对确定性的目标发布做了优化: 

leader_read
  -> follower_before_read
  -> target_compute
  -> SDK direct stream update
  -> latest camera frame fetch
  -> sample callbacks
  -> absolute 30 Hz scheduler sleep
领域 决策
从臂输出 使用 SDK direct joint stream。Studio 只发送最新目标；SDK 处理固定频率输出和超时行为。
从臂回读 不要在每个实时帧读取 follower_after。控制路径需要 follower_before 来做状态读取、裁剪和录制。诊断时可以通过 --follower-readback-every N 开启周期性回读。
摄像头 OpenCV 摄像头运行在后台 latest-frame 线程中。控制循环只获取缓存帧，因此摄像头 USB 或解码抖动不会阻塞目标发布。
调度器 按绝对 30 Hz tick 睡眠，并保留短保护窗口，因此普通 OS 睡眠超时不会累积成循环漂移。
Debug 指标 Debug 数据需要显式开启，并以阶段汇总形式报告。默认路径保持同样行为，但不存储逐阶段样本。

--debug 和 --follower-readback-every 的具体运行方式也放在数据集录制教程中，避免概念页与操作页维护两套命令。

读取 CLI 输出

字段 含义
iterations 已完成的 Studio 控制样本数。30 Hz 运行 60 秒时，健康结果应接近 1800。
target_hz 来自配置的 Studio 循环目标: configs/session.yaml。
observed_hz 实际 Studio 循环频率，计算方式为 iterations / elapsed_s。
last_latency_ms 最后一次循环步骤耗时，包括主臂读取、从臂动作前读取、目标计算、stream 更新、latest-frame 摄像头获取和回调。
max_latency_ms 运行中最慢的循环步骤。
elapsed_s 遥操作运行的墙钟时长。
phase_latency_ms 仅在使用 --debug 时出现；会报告每个遥操作阶段的次数、最近值、平均值、p50、p95 和最大延迟。 camera_read:* 是缓存帧获取成本， camera_age:* 表示帧新鲜度， scheduler_lag 是计时漂移。
