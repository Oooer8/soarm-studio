数据集录制

正式录制前先完成一次不写数据集的遥操作验证，确认主从方向、关节范围、夹爪和控制频率都正常，再开始采集 episode。

录制前验证遥操作

先完成机械臂标定并运行 check。清空从臂工作区，把主臂和从臂放在各自安全的中间姿态，然后运行 30 秒自由测试；--free-test 会在启动控制循环前再次执行就绪检查。

CLI 
soarm-studio teleop \
  --config configs/session.yaml \
  --free-test \
  --seconds 30

  1. 命令启动后握住主臂，依次缓慢移动肩、肘、腕关节，最后开合夹爪。每次只改变一个主要关节，观察从臂是否同向、连续且没有撞到软件限位；确认后再做组合动作。

  2. 如果从臂方向相反、突然跳变、接近机械硬限位或出现异常延迟，立即按 Ctrl-C，不要进入录制。先检查角色绑定、标定范围和 check 输出。

  3. 正常结束时状态应为 TELEOP_READY，observed_hz 应接近配置的 target_hz。通过后再录制数据。

需要时运行遥操作诊断

频率偏低或动作延迟时，先运行 debug；如果还要周期性读取命令后的从臂状态，再加入 --follower-readback-every 10。指标含义见遥操作输出。

CLI 
soarm-studio teleop \
  --config configs/session.yaml \
  --free-test \
  --seconds 60 \
  --debug

soarm-studio teleop \
  --config configs/session.yaml \
  --free-test \
  --seconds 60 \
  --debug \
  --follower-readback-every 10

开始录制数据集

录制会写入 LeRobot Dataset v3 的本地数据集，并附带描述硬件会话和同步质量的 SOARM Studio sidecar。先用一个短 episode 验证任务文本、画面、关节数据和输出目录，再扩大采集规模。

CLI 
soarm-studio record \
  --config configs/session.yaml \
  --episodes 1 \
  --warmup 1 \
  --seconds 10 \
  --task "pick object" \
  --overwrite

数采时序

录制时，实时循环只负责关节状态、action 和单调时间戳，图像同步在 episode 结束后处理。首个 episode 的 --warmup 和正式采集共用同一个 TeleopLoop，warmup 结束后不会重新 sync_start 或重启从臂 stream；支持 history 的相机从 episode 起点开始保存后台采集线程里的帧。保存数据集前，Studio 会为每个控制样本按 monotonic_time_ns 匹配最近的相机帧。 

camera threads:
  camera.read() @ camera.fps -> latest frame

warmup:
  same TeleopLoop, no sample/history collection

episode start:
  enable episode camera history

Studio loop @ loop_hz:
  leader_read || follower_before_read
  -> action
  -> SDK direct stream update
  -> append ControlSample(joints, action, timestamp)

after episode:
  sample.timestamp -> nearest camera frame timestamp
  -> write LeRobot frame and quality sidecars

因此，遥操作 debug 里的 camera_age:* 表示 latest-frame 预览缓存的新鲜度；录制质量里的相机时间差则表示后处理匹配后的样本到图像 offset。

默认录制只保存汇总质量指标。需要查看每个相机的 raw frame timestamp、相邻帧间隔和每个控制样本的匹配 offset 时，在录制命令后加 --debug；输出会写到 episodes/episode_000000/camera_timing.json。
