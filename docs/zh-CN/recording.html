数据集录制

正式录制前先完成一次不写数据集的遥操作验证，确认主从方向、关节范围、夹爪和控制频率都正常，再开始采集 episode。

录制前验证遥操作

先完成机械臂标定并运行 check。清空从臂工作区，把主臂和从臂放在各自安全的中间姿态，然后运行 30 秒自由测试；--free-test 会在启动控制循环前再次执行就绪检查。

CLI 
soarm-studio teleop \
  --config configs/session.yaml \
  --free-test \
  --seconds 30

  1. 命令启动后握住主臂，依次缓慢移动肩、肘、腕关节，最后开合夹爪。每次只改变一个主要关节，观察从臂是否同向、连续且没有撞到软件限位；确认后再做组合动作。

  2. 如果从臂方向相反、突然跳变、接近机械硬限位或出现异常延迟，立即按 Ctrl-C，不要进入录制。先检查角色绑定、标定范围和 check 输出。

  3. 正常结束时状态应为 TELEOP_READY，observed_hz 应接近配置的 target_hz。通过后再录制数据。

需要时运行遥操作诊断

频率偏低或动作延迟时，先运行 debug；如果还要周期性读取命令后的从臂状态，再加入 --follower-readback-every 10。指标含义见遥操作输出。

CLI 
soarm-studio teleop \
  --config configs/session.yaml \
  --free-test \
  --seconds 60 \
  --debug

soarm-studio teleop \
  --config configs/session.yaml \
  --free-test \
  --seconds 60 \
  --debug \
  --follower-readback-every 10

开始录制数据集

录制会写入 LeRobot Dataset v3 的本地数据集，并附带描述硬件会话和同步质量的 SOARM Studio sidecar。先用一个短 episode 验证任务文本、画面、关节数据和输出目录，再扩大采集规模。

CLI 
soarm-studio record \
  --config configs/session.yaml \
  --episodes 1 \
  --warmup 1 \
  --seconds 10 \
  --task "pick object" \
  --overwrite \
  --save-policy manual

加上 --save-policy manual 后，每个 episode 开始前都会等待确认。录制过程中按 d 提前结束当前 episode 并进入选择步骤，按 a 舍弃并立即重新录制这个 episode。选择步骤中，按 d 保存并继续下一集，按 a 舍弃并重新录制。

数采时序

录制以 Studio 的目标 loop tick 作为唯一时间轴。首个 episode 的 --warmup 和正式采集共用同一个 TeleopLoop，warmup 结束后不会重新 sync_start 或重启从臂 stream。warmup 会按 1 ms 分桶取稳定主峰，自动估计 joint 读取提前量，以及相机“收到帧”到“估计曝光时刻”的偏移；正式采集开始前，Studio 会根据 warmup 或上一集相机 history 选择第一个 loop tick，让 sample 0 尽量贴近下一张估计曝光帧。用户通常只需要保留 --warmup 1，不用手动配置额外 offset。 

camera threads:
  camera.read() @ camera.fps -> latest frame

warmup:
  same TeleopLoop
  collect joint request/receive timing
  collect camera receive intervals
  -> estimate joint_read_lead_ms
  -> estimate camera_receive_to_estimated_exposure_ms

episode start:
  enable episode camera history
  choose first loop tick from warmup or previous episode camera phase

Studio loop @ loop_hz:
  target_tick[n] = episode_start + n / loop_hz
  sleep until target_tick[n] - joint_read_lead
  leader_read || follower_before_read
  -> action
  -> SDK direct stream update
  -> append ControlSample(joints, action, target_tick)

after episode:
  camera receive timestamp -> estimated exposure timestamp
  target_tick -> nearest estimated exposure timestamp
  -> write LeRobot frame and quality sidecars

因此，遥操作 debug 里的 camera_age:* 表示 latest-frame 预览缓存的新鲜度；录制质量里的 max_camera_age_ms 表示“匹配到的估计曝光时刻”和目标 loop tick 的差值。普通 USB/UVC 相机没有硬件曝光时间戳，所以这里是软件估计，不是硬件同步。

默认录制只保存汇总质量指标。需要查看自动时序模型、joint 读取时间、每个相机的 raw receive timestamp、估计曝光时间和每个控制样本的匹配 offset 时，在录制命令后加 --debug；输出会写到 episodes/episode_000000/camera_timing.json。其中 timing_model 会包含 joint_read_lead_ms 和 camera_receive_to_estimated_exposure_ms。

camera_phase_alignment_ms.phase_wait_ms 是超过“最早可行首个 tick”的额外相位等待，不是完整的录制启动等待；完整等待还包含 joint read 提前量和 1 ms 调度余量。普通 USB/UVC 相机仍然自由运行，Studio 只是选择第一个 loop tick 更贴近下一张估计曝光帧，后处理的最近帧匹配仍然保留。多集录制时，第一集优先使用 warmup，相邻后续集会用上一集的 camera history 继续估计相位。
