标定原理

SOARM 标定把原始舵机编码器 tick 转换成有意义的关节角、安全软件限位和可复现数据。Studio 命令只负责路由工作流；物理采样和配置回写仍在 soarm-sdk。

为什么需要标定

每台装好的机械臂都会有细小机械差异：舵盘安装偏移、镜像关节方向、走线方式，以及与名义模型不完全一致的可用行程。舵机可以报告原始 tick，但在软件知道哪个物理姿态是零位、哪个方向为正之前，这个 tick 还不是稳定的关节角。

标定会把这层关系写入每台机械臂配置，后续命令就可以用弧度而不是原始硬件 tick 推理。

标定数据模型 text 
raw servo tick
  -> zero_tick + direction
  -> joint radians
  -> min_rad / max_rad safety envelope
  -> check, teleop, record, replay, diagnostics

记录哪些内容

零位 tick

已知物理零位姿态下的当前编码器读数。它是把当前位置转换成关节弧度的参考偏移。

关节方向

即使两台机械臂或关节存在镜像，也能让正向关节运动与模型保持一致的符号约定。

最小/最大限位

手动扫动得到的安全可用行程，会保存为用于检查和裁剪的关节空间限位。

诊断

标定前后的状态报告用于确认机械臂可访问，且保存的限位合理。

实时流程如何工作

  1. 运行 check ，先确认角色配置和总线可访问。
  2. 按提示把选中的机械臂摆到已知零位姿态。
  3. SDK 会把当前原始编码器 tick 采集为 zero_tick。
  4. 扭矩会被关闭，因此你可以手动扫动每个关节。
  5. SDK 会连续采样关节读数，并跟踪观测到的最小/最大值。
  6. 更新后的标定会写回对应机械臂配置。
  7. 运行 check ，然后再进入遥操作或录制。

为什么重要

遥操作依赖主臂和从臂读数使用同一套关节约定。录制依赖观测能跨会话复现。安全则依赖软件限位匹配桌面上的真实机械臂。错误标定可能让机械臂看起来已连接，但角度、限位或镜像符号是错的。

!
标定以读取为主

实时流程会在你手动移动机械臂时读取编码器位置。零位采集或范围扫描期间不应主动发送运动命令。
