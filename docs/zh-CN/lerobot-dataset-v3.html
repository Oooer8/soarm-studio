LeRobot Dataset v3

SOARM Studio 录制的是 LeRobot Dataset v3 的本地数据集。这个格式把用户看到的 episode 访问方式和底层存储拆开：低维状态和动作写入 Parquet，摄像头帧编码为 MP4，metadata 负责描述 schema、统计量、任务文本和每个 episode 在共享文件中的位置。

这页基于 Hugging Face LeRobotDataset v3.0 文档 以及 LeRobot 当前源码中的路径模板。

格式组成

部分 作用
表格数据 timestamp、observation.state、action 等逐帧数据写入 data/ 下的 Parquet 文件。
视觉数据 每个摄像头对应 observation.images.<camera>，视频写入 videos/ 下的 MP4 文件。
Metadata meta/info.json 保存 schema、fps、版本和路径模板；meta/stats.json 保存归一化统计；meta/episodes/ 保存 episode 长度、任务和文件偏移。

本地目录

SOARM Studio 当前写入一个简洁的本地 v3 布局，主要文件如下。随着 episode 增长，LeRobot v3 的设计允许把多段 episode 聚合到较大的 Parquet/MP4 shard 中；SOARM 目前保持单个本地 chunk 起步，便于检查和调试。 

datasets/soarm/
  meta/
    info.json
    stats.json
    tasks.parquet
    episodes/chunk-000/file-000.parquet
    soarm_session.json
    sync_quality.json
  data/chunk-000/file-000.parquet
  videos/observation.images.<camera>/chunk-000/file-000.mp4
  episodes/episode_000000/quality.json
i
当前 repo 的格式确认

src/soarm_studio/datasets/lerobot_v3/schema.py 使用 codebase_version=v3.0，并采用 LeRobot 当前源码中的 meta/tasks.parquet、meta/episodes/chunk-000/file-000.parquet、data/chunk-000/file-000.parquet 和 videos/{video_key}/chunk-000/file-000.mp4 路径模板。meta/soarm_session.json、meta/sync_quality.json 和 episode quality 文件是 SOARM 的额外 sidecar。

字段含义

字段 SOARM 含义
observation.state 动作发送前读取到的从臂关节位置，顺序来自 configs/session.yaml 的 joints。
action 当前帧发送给从臂 direct joint stream 的目标关节位置。
timestamp episode 内相对时间，第一帧为 0 秒，后续帧来自控制循环的 monotonic 时间。
task_index 任务文本在 meta/tasks.parquet 中的整数索引。
observation.images.* 启用的 wrist、third-person 等摄像头视频特征，实际帧存放在 videos/。
